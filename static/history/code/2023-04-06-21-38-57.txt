以下是找回密码活动图的UML代码，使用了菱形图标表示验证信息的过程：

```plantuml
@startuml
title 找回密码活动图

start
:请求找回密码;
if (验证信息?) then (true)
  -->[true]
  :发送邮件到绑定邮箱;
  :提示用户进入邮箱查看邮件;
  :进入邮箱;
  :点击邮件中的链接;
  if (输入新密码?) then (true)
    -->[true]
    :重置密码;
    :提示密码已重置;
  else (false)
    -->[false]
    :提示输入新密码;
    :重新填写新密码;
  endif
else (false)
  -->[false]
  :提示验证失败;
  :重新填写验证信息;
endif
stop

@enduml
```

这个活动图与之前的示例非常相似，但使用了菱形图标表示验证信息的过程。当用户请求找回密码后，如果验证信息成功，则系统会发送重置密码链接到绑定的邮箱中，并提示用户进入邮箱查看邮件。当用户确认了电子邮件中的链接并输入新密码时，系统将重置密码并提示用户密码已被重置。如果用户没有输入新密码，则系统将提示用户重新输入新密码。如果验证失败，则系统将提示用户重新输入验证信息。

该流程包括“请求找回密码”、“验证信息”、“发送邮件到绑定邮箱”、“提示用户进入邮箱查看邮件”、“进入邮箱”、“点击邮件中的链接”、“输入新密码”、“重置密码”、“提示密码已重置”、“提示输入新密码”、“提示验证失败”、“重新输入验证信息”等步骤。这个活动图描述了找回密码的完整过程，并使用了菱形图标表示验证步骤。
